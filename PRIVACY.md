# ProduTime Privacy Notice

Last updated: 2026-05-30

ProduTime is designed as a local-first desktop app.

## Default Local Mode

By default, ProduTime stores activity records, settings, reports, and app data
on the user's own device. This can include active application names, window
titles, idle periods, schedule settings, report history, and generated PDFs.

ProduTime does not send telemetry, activity records, usage analytics, or report
contents to George Karagioules.

## Optional Network Activity

Network activity may occur only when a user or administrator uses a networked
feature, including:

- checking for or downloading updates from GitHub Releases;
- opening external links;
- pairing the device with a ProduTime admin console;
- sending configured email alerts or reports; or
- connecting to an administrator-provided management endpoint.

When managed/admin features are used, data shared with that admin environment
depends on the administrator's configuration and may include device status,
policy state, activity summaries, or report-related data.

## Local Data Control

Reports and app data remain on the user's device unless the user or an
administrator exports, emails, syncs, or otherwise shares them through an
enabled feature.
